- [Auth cookie vs localStorage](auth-cookie-localstorage.md) — middleware baca cookie tapi auth store simpan ke localStorage saja → infinite redirect loop; fix: set cookie di login/loginDemo/logout/rehydrate, dan tambah `/` ke PUBLIC_PATHS middleware
- [Auth rehydrate timing fix](auth-rehydrate-timing.md) — 100+ halaman redirect ke dashboard karena rehydrate() dipanggil di useEffect (async); fix: pindah ke useState lazy initializer di ConditionalLayout agar sinkron sebelum render anak
- [No-login auto-login](no-login-autologin.md) — user minta hapus semua form login; solusi: autoLogin() di ConditionalLayout otomatis call /auth/login admin saat token kosong; token disimpan localStorage normal; 120+ halaman tidak perlu disentuh
- [JWT auth guard race fix](jwt-auth-race.md) — JwtAuthGuard adalah stub (return true, tidak set request.user) → RolesGuard 403; fix: JwtAuthGuard parse JWT manual pakai createRequire+jsonwebtoken; frontend: AuthInitializer panggil autoLogin, axios interceptor baca localStorage per-request, useDashboardData tunggu token di dependency array
- [Kledo data merge pattern](kledo-data-merge.md) — semua halaman sales merge data lokal + Kledo: Promise.allSettled([local, kledo]), dedup by kledoId/name, badge sumber; URL `kledo-{id}` untuk kontak/produk Kledo; tidak ada DEMO fallback
- [Kledo invoice push](kledo-invoice-push.md) — POST /kledo/invoices dengan {trans_date, due_date, ref_number, memo, contact_id, contact_name, discount, include_tax, items:[{product_id,name_item,qty,rate,discount,unit}]}; push setelah simpan lokal; kledoStatus: idle/syncing/ok/error
- [Kledo invoice quirks](kledo-invoice-quirks.md) — 3 gotcha: (1) finance_account_id harus pakai "product account" bukan COA (id 3234 valid); (2) due_date wajib ada; (3) contact_id wajib + contact tipe customer = type_id:3
- [Kledo contact search param](kledo-contact-search.md) — `?keyword=` di /finance/contacts TIDAK bekerja (selalu return kontak random); param yang benar adalah `?search=`
- [WA server-side sender](wa-server-sender.md) — server-side WA di lib/server/waServer.ts baca FONNTE_TOKEN/FONNTE_GROUP_INVOICE/FONNTE_GROUP_PAYMENT dari process.env; kirim ke 3 tujuan paralel; response berisi wa.grupOrder/wa.grupPayment/wa.konsumen
- [Local orders DB intercept](local-orders-db.md) — Next.js route intercept di app/api/sales/orders; jika DATABASE_URL tidak ada (CasaOS frontend container), forward langsung ke backend; jika ada (Replit), pakai local_orders table PostgreSQL
- [Backend integration Replit](backend-integration.md) — bcrypt native tidak bisa dicompile di Replit; ganti ke bcryptjs; auth.service.ts baca ADMIN_EMAIL+ADMIN_PASSWORD dari env var; wajib set agar cocok dengan frontend; PORT=3000 backend, BACKEND_URL=http://localhost:3000
